# **Spatial Expander Plugin — Revised Developer Specification**

**Version:** 1.1 (Post-Milestone 1)  
**Date:** 2026-06-15  
**Status:** Milestone 1 (3.1 Phantom Center) — COMPLETE

---

## 1. Objective
Real-time stereo-to-multichannel spatial expander (VST3/AU/AAX). Reproduces a cascading center-extraction workflow as a single internalized plugin. Extracts discrete spatial layers from stereo via successive phantom-center extraction passes inside one STFT frame. Targets music producers and immersive playback.

**Latency target:** ≤ 10.7 ms (Live/512), ≤ 21.3 ms (Standard/1024), ≤ 42.7 ms (Quality/2048).

---

## 2. Architecture
**Internalized Phantom-Center Cascade.** A single STFT frame performs multiple spectral extraction passes sequentially. Each pass extracts a correlated center from two input spectra using a minimum-magnitude phantom-center filter; the residual becomes a discrete output channel. The extracted center feeds into the next pass. No external VST chaining. No temporal tracking.

**Rejected:** per-bin routing, object-grouping STFT, ambisonics, multi-band filterbank, Wiener-filter amplitude extraction.

---

## 3. Layer-to-Channel Mapping

| Layers | Output Layout |
|--------|---------------|
| 1 | 3.1: C, L, R |
| 2 | 5.1: C, FrontL, FrontR, RearL, RearR |
| 3 | 7.1: C, FrontL, FrontR, SideL, SideR, RearL, RearR |
| 4 | 9.1: C, FrontL, FrontR, WideL, WideR, SideL, SideR, RearL, RearR |

**Plus LFE on all layouts.**

## 4. Signal Flow
```
Stereo Input (L, R)
    ↓
[STFT] — N-sample Hann window, 50% overlap
    ↓
Layer 1: PhantomCenterExtract(L, R) → C1, SideL, SideR
    ↓
Layer 2: PhantomCenterExtract(SideL, C1) → FrontL, RearL
          PhantomCenterExtract(SideR, C1) → FrontR, RearR
    ↓
Layer 3: PhantomCenterExtract(FrontL, RearL) → SideL2, FrontL2, RearL2
          PhantomCenterExtract(FrontR, RearR) → SideR2, FrontR2, RearR2
    ↓
Layer 4: PhantomCenterExtract(FrontL2, SideL2) → WideL, FrontL3, SideL3
          PhantomCenterExtract(FrontR2, SideR2) → WideR, FrontR3, SideR3
    ↓
[Auto-Calibration] — internal pink noise sweep, per-channel gain table
    ↓
[iSTFT per output channel] — each channel is a full, coherent spectrum
    ↓
[LFE] — LP80((L+R)/2), additive
    ↓
Multichannel Output
```

## 5. STFT Front End
- **Window:** Hann, 50% overlap
- **Hop size:** N/2
- **FFT Sizes:** 512 (Live), 1024 (Standard), 2048 (Quality)
- **Sample rates:** 44.1 / 48 / 96 kHz
- **Synthesis window:** Dual Hann (WOLA) — `w_synth[n] = w[n] / (w[n]² + w[n+N/2]²)`

### 5.1 Latency
The plugin reports **hop size (N/2)** samples of latency, **not** N and **not** 3N/2.

**Reasoning:** The first valid, perfectly reconstructed sample occurs at output position `hopSize`, which corresponds to input position `0`. The first `hopSize` samples are single-frame ramp-up and are skipped. All subsequent output is fully valid.

| Mode | Window | Hop | Latency @ 48 kHz |
|------|--------|-----|------------------|
| Live | 512 | 256 | 5.3 ms |
| Standard | 1024 | 512 | 10.7 ms |
| Quality | 2048 | 1024 | 21.3 ms |

### 5.2 Real-Time Safe Window Switching
The user may select window size via UI parameter. The change must:
1. Trigger on the **message thread** (no allocations in audio thread).
2. Mute output during the switch (10 ms fade out/in to prevent clicks).
3. Call `setLatencySamples(hopSize)` and `updateHostDisplay()` after the switch so the DAW re-queries latency.
4. Re-prepare the LFE extractor with the new latency so it stays sample-aligned.

**Note:** Some DAWs (Pro Tools, Logic) may not handle dynamic latency changes gracefully and may require a plugin reload. Reaper and Cubase handle it correctly.

### 5.3 Output Buffer Logic (Critical)
Frame processing and output consumption must be **decoupled** within each `processBlock()` call:

1. **Accumulate phase:** Write all input samples into the circular buffer. Process every frame that reaches the hop threshold.
2. **Output phase:** After all input is consumed, read all available output samples (`min(numSamples, outReady)`) in one contiguous block.
3. **Zero-fill phase:** Any remaining deficit (only during startup) is zeroed.

**Failure mode:** Interleaving frame processing with per-sample output causes 50% duty-cycle gaps in the audio stream, producing severe comb-filtering and "watery" artifacts that persist on summation.

## 6. Phantom Center Extraction Algorithm (Corrected)

For each STFT bin `k`, given complex spectra `A[k]` and `B[k]`:

```
aMag = |A[k]|
bMag = |B[k]|
cMag = min(aMag, bMag)

C[k] = 0
if cMag > 1e-18:
    sum = A[k] + B[k]
    sumMag = |sum|
    if sumMag > 1e-18:
        C[k] = cMag * (sum / sumMag)

Res_A[k] = A[k] - C[k]
Res_B[k] = B[k] - C[k]
```

### 6.1 Why This Works

**Panning behavior:**
- **Mono center (A = B):** `cMag = |A|`, `C = A`. Residuals are zero. Energy goes 100% to Center.
- **50% left (A = 0.75S, B = 0.25S):** `cMag = 0.25`, `C = 0.25S`. `Res_A = 0.50S`, `Res_B = 0`. Energy splits between Center and Left residual. Right residual is silent.
- **100% left (A = S, B = 0):** `cMag = 0`, `C = 0`. `Res_A = S`, `Res_B = 0`. Energy goes 100% to Left residual.

**Lossless reconstruction:** `2·C + Res_A + Res_B = A + B` (exact, algebraically guaranteed regardless of how `C` is computed).

**True separation:** `C` is fully subtracted from both inputs. Phase-inverting `C` and adding to `A` yields `Res_A` exactly.

**Anti-correlated content:** If `A = S` and `B = -S`, then `sum = 0`, `C = 0`. Center is silent. Correct behavior for out-of-phase stereo effects.

### 6.2 Rejected: Wiener Filter
The original specification proposed a Wiener filter:
```
h = (A * conj(B)) / (|B|² + epsilon)
```
This was **abandoned** because it produces unbounded amplification (`|h|` can reach 9× or higher) when one channel is significantly louder than the other. This causes:
- Meter explosions during panning.
- Volume jumps and perceived lag.
- Phase instability.

The minimum-magnitude extractor replaces it entirely.

### 6.3 Rejected: IIR Temporal Smoothing
Smoothing the filter coefficient `h` across frames with a first-order IIR:
```
h_smooth = alpha * h + (1-alpha) * h_smooth_prev
```
was **abandoned** because it creates:
- Panning lag (~50 ms time constant at alpha=0.3).
- Volume explosions when `h` transitions from large to small values.
- Unstable behavior at pan boundaries.

The phantom-center extractor does not require temporal smoothing. Per-frame extraction is already stable because `|C|` is bounded by the weaker input channel.

## 7. Cascade Structure

### 3.1 (1 layer)
```
C1, SideL, SideR = PhantomCenterExtract(L, R)
Outputs: C1, SideL, SideR
```

### 5.1 (2 layers)
```
C1, SideL, SideR = PhantomCenterExtract(L, R)
FrontL, RearL = PhantomCenterExtract(SideL, C1)
FrontR, RearR = PhantomCenterExtract(SideR, C1)
Outputs: C1, FrontL, FrontR, RearL, RearR
```

### 7.1 (3 layers)
```
C1, SideL, SideR = PhantomCenterExtract(L, R)
FrontL, RearL = PhantomCenterExtract(SideL, C1)
FrontR, RearR = PhantomCenterExtract(SideR, C1)
SideL2, FrontL2, RearL2 = PhantomCenterExtract(FrontL, RearL)
SideR2, FrontR2, RearR2 = PhantomCenterExtract(FrontR, RearR)
Outputs: C1, FrontL2, FrontR2, SideL2, SideR2, RearL2, RearR2
```

### 9.1 (4 layers)
```
... (same as 7.1 through layer 3)
WideL, FrontL3, SideL3 = PhantomCenterExtract(FrontL2, SideL2)
WideR, FrontR3, SideR3 = PhantomCenterExtract(FrontR2, SideR2)
Outputs: C1, FrontL3, FrontR3, WideL, WideR, SideL3, SideR3, RearL2, RearR2
```

### 7.1 Phase Cancellation Policy
- **7.1 and 9.1:** Full phase cancellation. After extracting `SideL2` from `(FrontL, RearL)`, the output residuals `FrontL2` and `RearL2` have `SideL2` fully subtracted. Same for all layers.
- **5.1:** The user may optionally skip phase cancellation on rear channels for better side phantom imaging. For the plugin, implement full cancellation consistently; the user can adjust via calibration gains if needed.

## 8. Automatic Calibration

**Purpose:** Maintain constant perceived loudness as a mono source pans through the layers.

**Method:** Internal pink noise sweep (no audible output).

**Procedure:**
1. Generate stereo pink noise internally (not output to speakers).
2. Sweep pan from hard-left to hard-right in 1 dB ILD steps.
3. At each step, run the full cascade and measure total energy across all output channels.
4. Compute per-channel gain corrections so that total energy varies by < ±0.5 dB across all pan positions.
5. Store gains as a lookup table indexed by ILD region.

**Calibration trigger:** Run once at plugin initialization, and again when user changes:
- Number of layers
- Window size
- Output format

**Duration:** ~50–100 ms.

**Playback safety:** If calibration triggers during active DAW playback, the plugin must **mute all output** until calibration completes. The mute/unmute transition must use a 10 ms fade to prevent clicks.

**Resulting gain table (example, to be computed automatically):**

| Channel | Typical Gain |
|---------|-------------|
| C | 0 dB |
| Front L/R | 0 dB |
| Side L/R | +6 dB |
| Wide L/R | +6 dB |
| Rear L/R | 0 dB |

**User override:** `Channel Trim` — per-channel ±12 dB offset applied after auto-calibration.

---

## 9. LFE
`LFE = LP80((L+R)/2)`. Extracted before STFT. Mixed additively with ground channels after iSTFT.

**Latency alignment:** The LFE extractor must report the same total latency as the STFT (hop size). It achieves this via a FIR low-pass filter plus a compensating delay line sized to `stftLatency - firDelay`.

## 11. Acceptance Criteria

### Functional
- Mono sine wave (L = R) → **only Center**, all other channels silent.
- Hard-panned left (L only) → **only Left residual** (SideL at layer 1, RearL at layer 2, WideL at layer 4).
- Re-summing all channels in the **multichannel domain** with correct weights recovers original stereo exactly (lossless).
- Latency matches hop size (N/2).

### Perceptual
- No watery/MP3-like artifacts when all channels are active.
- No comb filtering or phasiness.
- Mono source panned through layers maintains constant loudness (±0.5 dB after calibration).
- Each channel sounds like a complete, full-spectrum signal.

### Technical
- Real-time safe (no allocations in audio thread).
- CPU usage < 10% on modern CPU at 48 kHz / 1024 samples.

### Null-Test Protocol
**Important:** The null test must be performed in the **multichannel domain** or via a **manual downmix** (Center at +6 dB, L/R at 0 dB, all others muted). DAW auto-downfold matrices (e.g., ITU-R BS.775) typically attenuate the center channel by -3 dB, which will leave residual center-only content in the null test. This is a DAW behavior, not a plugin bug.

---

## 12. Known Limitations

- Dense, uncorrelated content (rain, crowd) may not extract into clean layers and may spread across multiple channels.
- Very short windows (512) reduce extraction precision and may allow slight center leakage into residuals.
- Solo individual channels sound "watery" or thin in isolation (inherent STFT property), but cancel perfectly when all channels are summed.
- Some DAWs do not handle dynamic latency changes gracefully when switching window sizes.
