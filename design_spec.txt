# **Spatial Expander Plugin — Developer Specification**

## 1. Objective
Real-time stereo-to-multichannel creative spatial expander (VST3/AU/AAX). Converts stereo into 7.1, 9.1, or 7.1.4/9.1.4 by stretching the original stereo field to a wider speaker ring. Targets music producers and immersive playback.

**Latency target:** ≤ 10.7 ms.

---

## 2. Rationale for the Stretch

When a mixer pans a source hard left in a stereo mix, they are not specifying an absolute angle of −30°. They are specifying *"this source should occupy the maximum leftward extent available in this playback system."* On headphones, that extent is inside the left ear. On stereo speakers, it is the left speaker at −30°. In a 360° ring, the maximum leftward extent is the rear-left speaker at −150°.

The stretch is therefore a **geometric rescaling** that preserves the relational structure of the stereo field (center is center, moderate-left is moderate-left, extreme-left is extreme-left) while adapting that structure to a new speaker topology. It is analogous to an anamorphic lens stretching a 16:9 image to fill a 2.39:1 screen: relative proportions are preserved; absolute geometry is adapted to the display.

**Architecture:** Object-Grouping STFT. A single short STFT frame groups adjacent frequency bins into coherent "objects" based on spatial similarity, then routes each object as a whole via a tent-map panning law.

**Rejected:** cascading extraction (latency stacks), ambisonics (cannot solo at speaker angles), per-bin STFT (destroys coherence), multi-band IIR filterbank (inter-band delay, coarse resolution).

---

## 3. Signal Flow
```
Stereo Input (L, R)
    ↓
[LFE Extraction] — LP80((L+R)/2) → LFE channel (bypasses spatial routing)
    ↓
[STFT] — 512-sample Hann window, 50% overlap
    ↓
Per-bin analysis (ILD, correlation, phase difference, spectral flux)
    ↓
Object grouping (adaptive thresholds)
    ↓
Post-grouping harmonic merge
    ↓
Height decision (post-grouping, with transient boost)
    ↓
Stretched azimuth estimation (ILD → stretched angle)
    ↓
Tent-map panning per object
    ↓
Front phantom center blend (optional, see Section 9)
    ↓
Temporal smoothing (birth/death ramps + gain smoothing)
    ↓
[iSTFT per channel]
    ↓
Multichannel Output (ground + height + LFE)
```

---

## 4. STFT Front End
- **Window:** 512-sample Hann, 50% overlap
- **Sample rates:** 44.1 / 48 / 96 kHz (auto-scaled)
- **Latency:** 10.7 ms (full window buffer at 48 kHz; causal implementation requires complete 512-sample frame before processing)

---

## 5. Per-Bin Analysis
For each bin `k`:
- `ILD[k] = 20·log₁₀(|L[k]| / |R[k]|)`
- `Correlation[k] = |L[k]·R[k]*| / (|L[k]|·|R[k]|)`
- `PhaseDiff[k] = unwrap(arg(L[k]) − arg(R[k]))`
- `SpectralFlux[k] = |Magnitude[k] − MagnitudePrev[k]|`

---

## 6. Object Grouping

**Initial scan** (low-to-high frequency):
```
start_idx = 0
for k = 1 to N/2:
    ild_diff = |ILD[k] − ILD[k−1]|
    corr_diff = |Correlation[k] − Correlation[k−1]|
    
    if Correlation[k−1] > 0.9:   ild_thresh = 4.0 dB
    elif Correlation[k−1] < 0.5: ild_thresh = 1.0 dB
    else:                         ild_thresh = 2.5 dB
    
    corr_thresh = 0.2
    
    if (ild_diff < ild_thresh) AND (corr_diff < corr_thresh):
        continue  // same object
    else:
        finalize object at k−1
        start_idx = k
```

**Post-grouping harmonic merge:**
```
for each pair of adjacent objects (A, B):
    ratio = B.spectralCentroid / A.spectralCentroid
    if (|ratio − round(ratio)| < 0.15) AND (A.meanCorr > 0.8) AND (B.meanCorr > 0.8):
        merge B into A
```

**Note:** Non-harmonic dense textures (detuned synths, orchestral clusters, thick pad layers) may still fragment despite harmonic merge. The 64-slot object pool limits the damage by capping the number of active objects.

---

## 7. Height Decision (Post-Grouping)

For each object:
```
energyAbove = sumEnergy(object, freq > heightThreshold)
transientEnergy = sumSpectralFlux(object)
effectiveAbove = energyAbove + (transientEnergy * transientBoost * 0.5)

if (Height Split == ON) AND (0.25 < effectiveAbove/totalEnergy < 0.75):
    // Object is genuinely split across the spectral boundary
    // Split into two objects at heightThreshold frequency
    // Route low portion to ground, high portion to height
    // Both portions use the same stretched azimuth
else:
    if (effectiveAbove / totalEnergy) > 0.5:
        targetLayer = HEIGHT
        route entire object to height speaker at object's stretched azimuth
        remove from ground (subtractive)
    else:
        targetLayer = GROUND
        route entire object to ground via tent-map
```

**Height speakers (all at +30° elevation):**
- Ltf (−30°), Rtf (+30°), Ctr (0°), Ltr (−150°), Rtr (+150°)

**Format channel counts:**
- **7.1.4:** Ltf, Rtf, Ltr, Rtr (4 height speakers)
- **9.1.4:** Ltf, Rtf, Ctr, Ltr, Rtr (5 height speakers)

**Note:** The height layer is purely spectral. Bright snare transients, high-frequency synth leads, or other non-overhead content above the threshold will route to height. This is a known creative effect, not a spatial extraction.

---

## 8. Stretched Azimuth Estimation

The original stereo field spans ±30°. This is stretched to a full 360° ring via a smooth monotonic curve calibrated to the following anchor points:

| |ILD| (dB) | Stretched Angle (°) |
|--------|---------------------|
| 0 | 0 |
| 3 | 30 |
| 8 | 90 |
| 15 | 150 |
| 30 | 180 |

**Implementation:** A 5-point cubic spline with continuous first and second derivatives (C2), or an equivalent smooth parametric function that passes through all anchor points. The curve must be monotonic and saturate at 180°.

**Stretch parameter:** User-adjustable 0.0–1.0 (default 1.0). At 0.0, `stretchedAngle = originalAngle` (±30° stereo field, no stretch). At 1.0, full curve. Intermediate values linearly interpolate between original and full stretch:

```
finalAngle = (1 - stretch) * originalAngle + stretch * stretchedAngle
```

where `originalAngle = sign(ILD) * min(30, abs(ILD) * 10)`.

**Rear bias:** If `|PhaseDiff − 180°| < 30°` AND `|ILD| < 6 dB` AND `Correlation < 0.5`:
`finalAngle += sign(ILD) · rearBias · 60°`

---

## 9. Tent-Map Panning Law

**7.1 ground speakers:** Lb(−150°), Ls(−90°), L(−30°), C(0°), R(+30°), Rs(+90°), Rb(+150°)  
**9.1 adds:** Lw(−60°), Rw(+60°)

```
Given object finalAngle θ:
1. Find nearest speakers (spkA at θA, spkB at θB)
2. frac = (θ − θA) / (θB − θA)

if frac < soloZone (default 0.05):
    gain[spkA] = 1.0
elif frac > (1 − soloZone):
    gain[spkB] = 1.0
else:
    gain[spkA] = 1 − frac
    gain[spkB] = frac
```

**Cosine rolloff at solo-zone edges:** To eliminate mathematical discontinuities, the gain at the boundary uses a smooth cosine transition:

```
if frac < soloZone:
    gain[spkA] = 0.5 + 0.5 * cos(π * frac / soloZone)
    gain[spkB] = 1.0 - gain[spkA]
elif frac > (1 - soloZone):
    gain[spkB] = 0.5 + 0.5 * cos(π * (1.0 - frac) / soloZone)
    gain[spkA] = 1.0 - gain[spkB]
```

**Optional micro-bleed:** For additional smoothness on slow pans, a constant −20 dB bleed to the next-nearest speaker can be added. This is not implemented by default but can be exposed as a "Smoothness" parameter if user testing reveals stepping artifacts.

---

## 10. Front Phantom Center

**Parameter:** `Front Phantom Center` — checkbox, default **unchecked**.

**Behavior:**
- **Unchecked (default):** Center objects route to **C only** (solo). The tent-map handles center objects normally.
- **Checked:** When an object routes to C, a portion of its energy is blended to **L and R** to create a phantom center image across the front three speakers.

**Implementation:**
```
if (FrontPhantomCenter == ON) AND (centerGain > 0):
    phantomWidth = 0.25
    gain[C] = centerGain * (1.0 - phantomWidth)
    gain[L] += centerGain * phantomWidth * 0.5
    gain[R] += centerGain * phantomWidth * 0.5
    // Total energy preserved: 0.75 + 0.125 + 0.125 = 1.0
```

---

## 11. Temporal Continuity

**Frame-to-frame object tracking:** Match objects using ILD difference (40%), spectral centroid difference (35%), bandwidth difference (25%). Inherit persistent ID if match score < threshold; otherwise assign new ID.

**Fixed-size object pool:** 64 slots maximum. New objects fill the first dead slot. If all slots are full, the weakest object is overwritten. This is lock-free and allocation-free.

**Gain smoothing:** `smoothedGain[n] = α · targetGain[n] + (1−α) · smoothedGain[n−1]` where `α = 1 − exp(−1 / (τ · fs))`, τ = 20–50 ms (default 30 ms).

**Hysteresis:** Once assigned to a speaker, require 2× stronger ILD shift to change assignment.

**Birth/death ramping:**
```
if object.isNew:
    ramp = min(1.0, timeSinceBirth / 20ms)
    smoothedGain = targetGain * ramp
elif object.isDying:
    ramp = max(0.0, 1.0 − timeSinceDeath / 20ms)
    smoothedGain = previousGain * ramp
```

---

## 12. LFE Channel
Extracted **before** STFT from original time-domain input: `LFE = LP80((L+R)/2)`. Mixed additively with ground channels after iSTFT.

**Implementation note:** Ensure the LFE lowpass filter has the same group delay as the STFT analysis path, or compensate for the difference in the output stage to prevent low-frequency smearing relative to the spatialized content.

## 16. Acceptance Criteria

**Functional**
- Mono source (L = R) → **only Center** (or L/C/R phantom if Front Phantom Center is ON).
- Hard-panned left (ILD > +15 dB) → **only Rear Left (Lb)**.
- Moderately left (ILD ~ +8 dB) → **only Side Left (Ls)**.
- Slightly left (ILD ~ +3 dB) → **only Front Left (L)**.
- Between adjacent speakers → **only those two adjacent speakers** fire with smooth phantom image.
- Stretch = 0.0 → output identical to original stereo mapping (L/R only, no rear/side).
- High-frequency object above threshold → routed to **height** at same stretched azimuth, ground spectrally reduced.
- Latency ≤ 10.7 ms at 48 kHz with 512-sample window.

**Perceptual**
- Discrete sources maintain timbral coherence — no comb filtering or phasiness.
- Spatial image does not flicker near speaker boundaries.
- Height channels sound spectrally distinct from ground.
- No vertical phantom images between ground and height.

**Technical**
- Real-time safe (no allocations in audio thread).
- Supports standard DAW surround/Atmos channel routing.

---

## 17. Known Limitations

- Two sources panned close together may be fused into one object and routed to a compromised position.
- Single sources with strong room reflections may fragment if harmonicity override fails.
- Non-harmonic dense textures (orchestral clusters, detuned pads) may fragment into multiple micro-objects, creating a grainy spatial texture.
- Height layer is a creative psychoacoustic effect, not spatial extraction. Bright non-overhead content (snare transients, synth leads) above the threshold will route to height.
- Stereo recordings contain no true height information.
- Uncorrelated content (rain, crowd, dense reverb) may produce a grainy spatial texture due to rapid object fragmentation. This is an intentional trade-off to preserve the original stereo field's spatial relationships rather than collapsing them into a uniform surround bed.

---

## 18. Implementation Notes

**iSTFT optimization:** Running separate inverse transforms for 11–14 channels is computationally expensive. Where possible, share the inverse FFT computation and apply per-channel gains in the time domain, or use a single inverse transform into a multichannel buffer with per-channel overlap-add.

**LFE delay alignment:** The LFE extraction path must have matched group delay with the STFT path. If using a causal IIR lowpass for LFE, compensate for its delay in the output mixer or use a linear-phase FIR with delay equal to the STFT window.

## Critical Path & Risk Notes

1. **Object Grouping is the bottleneck.** If the adaptive thresholds in Section 6 fragment too aggressively on dense orchestral pads or detuned synths, the spatial image becomes "grainy." Budget time for threshold tuning.
2. **LFE alignment is subtle.** If you use a causal IIR for the LP80, measure its group delay precisely and delay-compensate the spatialized path. Do not guess.
3. **Height is creative, not extractive.** Do not spend cycles trying to "improve" height extraction with HRTF or ambisonic tricks—the spec explicitly rejected ambisonics. Stick to the spectral threshold + transient boost model.
4. **Stretch = 0.0 is a test mode.** Ensure the code path literally maps to L/R only with no rear/side energy. This is an acceptance gate that is easy to break accidentally when optimizing the panning law.
