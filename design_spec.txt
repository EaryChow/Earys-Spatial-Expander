# **Phantom Center Extraction Algorithm — Developer Specification**

## 1. What This Algorithm Does

Takes a stereo pair `(L, R)` and decomposes it into three full-bandwidth, timbrally coherent channels:

- **Center (C):** The correlated component shared by both channels.
- **Left Residual (Lᵣₑₛ):** Everything in the left channel that is not in the center.
- **Right Residual (Rᵣₑₛ):** Everything in the right channel that is not in the center.

**The two non-negotiable properties:**
1. **True separation:** If you phase-invert `C` and add it to `L`, you get `Lᵣₑₛ` with no audible center leakage. Same for `R`.
2. **Lossless reconstruction:** `2·C + Lᵣₑₛ + Rᵣₑₛ = L + R` exactly (within floating-point precision).

This is achieved with a single STFT frame and a per-bin Wiener-like extraction. No temporal tracking, no object grouping, no per-bin routing.

---

## 2. STFT Framework

| Parameter | Value | Notes |
|-----------|-------|-------|
| **Window** | Hann | `w[n] = 0.5 · (1 – cos(2πn/N))` |
| **Overlap** | 50% | Hop size = `N/2` |
| **FFT Size** | `N` (no zero-padding) | Same as window size |
| **Latency** | `N` samples | e.g., 21.3 ms @ 48 kHz for `N=1024` |
| **Sample Rates** | 44.1 / 48 / 96 kHz | Host-determined |

### 2.1 Perfect Reconstruction (Critical)
With Hann + 50% overlap, the **analysis** windows satisfy COLA (sum to 1.0).  
However, because we modify spectra and then window again for synthesis, we need the **synthesis** window to compensate for the squared overlap:

**Synthesis window:**
```
w_synth[n] = w[n] / (w[n]² + w[n + N/2]²)
```

For Hann, the denominator simplifies to `0.5 · (1 + cos²(2πn/N))`.  
Using `w_synth` guarantees that after overlap-add, the original signal is recovered exactly if the spectrum is unmodified.

**Implementation note:** Precompute `w_synth[n]` at initialization. Do not compute it in the audio thread.

## 3. The Core Extraction Step (Wiener Filter)

For each STFT bin `k` (0 ≤ k < N/2+1), given complex spectra `A[k]` and `B[k]`:

### 3.1 The Math

```
h[k] = (A[k] · conj(B[k])) / (|B[k]|² + ε)

C_A[k] = h[k] · B[k]
C_B[k] = conj(h[k]) · A[k]

C[k]   = (C_A[k] + C_B[k]) / 2
Res_A[k] = A[k] – C[k]
Res_B[k] = B[k] – C[k]
```

### 3.2 Epsilon (Regularization)

Use a **relative epsilon** to prevent division-by-zero while preserving the filter's behavior across dynamic ranges:

```
ε = 1e-6 · |B[k]|²
```

If `|B[k]|²` is extremely small (e.g., hard-panned source), `ε` dominates and `h[k] → 0`, correctly yielding `C[k] ≈ 0`.

**Alternative (simpler):** Fixed `ε = 1e-12` for float32, `1e-15` for float64. Both work; the relative version is safer for real-world audio.

### 3.3 Why This Works (Proof of Properties)

**Lossless reconstruction (algebraic guarantee):**
```
2·C + Res_A + Res_B
= 2·C + (A – C) + (B – C)
= A + B
```
This is exact regardless of the value of `h` or `ε`. It is impossible to break.

**True separation:**
By definition, `Res_A = A – C`. The center component `C` has been fully subtracted from `A`. If you invert `C` and add it to `A`, you get `A – C = Res_A`. There is no "leakage" because subtraction is exact.

**Mono source (A = B):**
```
h = |A|² / (|A|² + ε) ≈ 1
C = (1·A + 1·A)/2 = A
Res_A = A – A = 0,  Res_B = 0
```
Mono energy goes 100% to Center.

**Hard-panned left (B = 0):**
```
h = 0,  C = 0,  Res_A = A,  Res_B = 0
```
Energy stays 100% in the left residual.

**Panned source (e.g., A = 0.866·S, B = 0.5·S):**
```
C ≈ 1.183·S,  Res_A ≈ –0.317·S,  Res_B ≈ –0.683·S
```
The "negative" residuals are simply phase-inverted, attenuated copies of the source. Phase inversion is inaudible in a single channel. When all channels are summed with the correct reconstruction weights, they cancel perfectly and recover the original stereo.

## 4. Single-Layer Plugin (3.1 Mode) — First Milestone

This is your "phantom center VST replica." It implements exactly one extraction pass.

### 4.1 Signal Flow

```
Stereo Input (L, R)
    │
    ├──→ [STFT] ──→ L_fft, R_fft
    │                  │
    │                  ▼
    │           WienerExtract(L_fft, R_fft)
    │                  │
    │        ┌────────┼────────┐
    │        ▼        ▼        ▼
    │      C_fft   Lres_fft  Rres_fft
    │        │        │        │
    │        ▼        ▼        ▼
    │     [iSTFT]  [iSTFT]  [iSTFT]
    │        │        │        │
    │        ▼        ▼        ▼
    │      C_out   L_out    R_out
    │
    └──→ [LFE Filter] ──→ LFE_out
```

### 4.2 Per-Frame Processing

For each audio block:
1. **Buffer:** Write incoming samples into a circular buffer. When `N` new samples + `N/2` overlap are available, extract one frame.
2. **STFT:** Window the frame with `w[n]`, FFT to get `L_fft[k]`, `R_fft[k]`.
3. **Extract:** Apply the Wiener formula (§3.1) with `A = L_fft`, `B = R_fft`.
   - `C_fft[k] = C[k]`
   - `Lres_fft[k] = Res_A[k]`
   - `Rres_fft[k] = Res_B[k]`
4. **iSTFT:** For each output channel, inverse-FFT, window with `w_synth[n]`, overlap-add into the output circular buffer.
5. **LFE:** In the time domain, `LFE = LP80((L+R)/2)`. Mix additively after iSTFT.

### 4.3 LFE Filter

- **Type:** 4th-order Butterworth low-pass
- **Cutoff:** 80 Hz (configurable 40–200 Hz)
- **Implementation:** Bilinear transform from analog prototype. Two cascaded 2-pole sections (Direct Form I or II).
- **Extracted before STFT**, mixed additively after iSTFT.

## 5. Artifact Mitigation (Watery Sound)

Individual channels will sound "watery" or thin in isolation — this is inherent to any STFT-based extraction because adjacent bins are modified independently. **However, when all channels are summed, the artifacts cancel exactly because the process is linear and lossless.**

To minimize the watery artifact in isolation (for solo channel monitoring):

### 5.1 Temporal Smoothing (Recommended)
Apply a first-order IIR filter to the complex filter coefficient `h[k]` across consecutive frames:

```
h_smooth[k, t] = α · h[k, t] + (1 – α) · h_smooth[k, t–1]
```

- `α = 0.3` is a good starting point (range: 0.1 to 0.5).
- This reduces rapid frame-to-frame fluctuations, which are the primary cause of the "swimming" artifact.
- **Preserve the phase:** Smooth the complex `h` directly, not just its magnitude.

### 5.2 Frequency Smoothing (Optional)
Apply a 3-bin moving average to the magnitude of `h`:

```
|h_smooth|[k] = (|h|[k–1] + 2·|h|[k] + |h|[k+1]) / 4
```

Then reconstruct: `h_smooth[k] = |h_smooth|[k] · exp(j · arg(h[k]))`.

This prevents spectral discontinuities between adjacent bins.

### 5.3 Why Artifacts Cancel on Summation
The extraction is a **linear time-varying filter**. Any artifact introduced into `C` is also introduced into `Res_A` and `Res_B` with opposite phase. When you sum `2·C + Res_A + Res_B`, the artifacts sum to zero, leaving only the original signal.

## 6. Full Cascade (5.1 / 7.1 / 9.1)

For the full Spatial Expander, the single extraction is cascaded. All extractions happen **inside the same STFT frame** — no additional latency.

### 6.1 Internal Signal Definitions

```
Layer 1:
  C1      = WienerCenter(L, R)
  SideL   = L – C1
  SideR   = R – C1

Layer 2 (Left):
  FrontL  = WienerCenter(SideL, C1)
  RearL   = SideL – FrontL

Layer 2 (Right):
  FrontR  = WienerCenter(SideR, C1)
  RearR   = SideR – FrontR

Layer 3 (Left):
  SideL2  = WienerCenter(FrontL, RearL)
  FrontL2 = FrontL – SideL2
  RearL2  = RearL – SideL2

Layer 3 (Right):
  SideR2  = WienerCenter(FrontR, RearR)
  FrontR2 = FrontR – SideR2
  RearR2  = RearR – SideR2

Layer 4 (Left):
  WideL   = WienerCenter(FrontL, SideL2)   ← Uses original FrontL, not FrontL2
  FrontL3 = FrontL – WideL
  SideL3  = SideL2 – WideL

Layer 4 (Right):
  WideR   = WienerCenter(FrontR, SideR2)
  FrontR3 = FrontR – WideR
  SideR3  = SideR2 – WideR
```

### 6.2 Output Channel Mapping

| Layout | Output Channels |
|--------|-----------------|
| **3.1** | `C1`, `SideL`, `SideR` |
| **5.1** | `C1`, `FrontL`, `FrontR`, `RearL`, `RearR` |
| **7.1** | `C1`, `FrontL2`, `FrontR2`, `SideL2`, `SideR2`, `RearL2`, `RearR2` |
| **9.1** | `C1`, `FrontL3`, `FrontR3`, `WideL`, `WideR`, `SideL3`, `SideR3`, `RearL2`, `RearR2` |

### 6.3 Phase Cancellation Policy

- **7.1 and 9.1:** Full phase cancellation. After extracting `SideL2` from `(FrontL, RearL)`, the output residuals `FrontL2` and `RearL2` have `SideL2` fully subtracted. Same for all layers.
- **5.1:** The user may optionally skip phase cancellation on the rear channels for better side phantom imaging. For the plugin, implement full cancellation consistently; the user can adjust via calibration gains if needed.

### 6.4 Lossless Reconstruction Weights

To recover the original stereo exactly, sum the output channels with these weights:

| Layout | Channels at 2× | Channels at 1× |
|--------|---------------|----------------|
| **3.1** | `C1` | `SideL`, `SideR` |
| **5.1** | `C1` | `FrontL`, `FrontR`, `RearL`, `RearR` |
| **7.1** | `C1`, `SideL2`, `SideR2` | `FrontL2`, `FrontR2`, `RearL2`, `RearR2` |
| **9.1** | `C1`, `WideL`, `WideR` | `FrontL3`, `FrontR3`, `SideL3`, `SideR3`, `RearL2`, `RearR2` |

**Verification:**  
`Σ (weight · channel) = L + R` exactly.

## 7. Auto-Calibration (for Full Plugin)

### 7.1 Purpose
Extracted intermediate channels (Side, Wide) are perceptually quieter than expected. Auto-calibration ensures a mono source maintains constant loudness (±0.5 dB) as it pans through the layers.

### 7.2 Procedure
1. **Generate** stereo pink noise internally (not routed to output).
2. **Sweep** pan from hard-left to hard-right in 1 dB ILD steps.
3. **Measure** total energy across all output channels at each step.
4. **Compute** per-channel gain corrections so total energy varies < ±0.5 dB.
5. **Store** gains in a lookup table indexed by ILD region.

### 7.3 Playback Safety
- If calibration triggers during DAW playback, **mute all output** until complete.
- Use a **10 ms fade in/out** to prevent clicks.

### 7.4 User Trim
After auto-calibration, apply a per-channel trim: **±12 dB** offset.

## 8. Implementation Notes for the Developer

### 8.1 Real-Time Safety
- **No allocations** in the audio thread. Pre-allocate all FFT buffers, circular buffers, and gain tables at initialization.
- Use **lock-free atomic updates** for parameter changes (window size, layer count).
- If the user changes window size or layer count, trigger a new calibration on the next buffer cycle (with mute).

### 8.2 Memory Layout (Per Channel)
- **Input circular buffer:** `2 × N` samples (double-buffered for overlap)
- **FFT complex buffer:** `N` complex values (interleaved or planar)
- **Output circular buffer:** `2 × N` samples per output channel

For the first milestone (3.1 + LFE): 4 output channels = 4 × `2N` samples.

### 8.3 FFT Library
- **Windows:** Intel IPP / FFTW
- **macOS:** Apple vDSP (highly optimized)
- **Linux:** FFTW or PFFFT

Use real-to-complex FFT (r2c) and complex-to-real (c2r). Only process bins `0` to `N/2` (the rest are conjugate-symmetric for real inputs).

### 8.4 CPU Optimization
- The per-bin Wiener math is trivial (`O(N)`). The FFT dominates.
- For the full cascade (9.1), you perform multiple extractions per bin but still only **2 FFTs and 9 iFFTs** per frame. The extractions are just complex multiplies/adds — easily vectorized with SSE/AVX/NEON.

### 8.5 Numerical Precision
- Use **float32** for the audio path. It is sufficient.
- Use **float64** for the calibration energy accumulation to prevent drift.
